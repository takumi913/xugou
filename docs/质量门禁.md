# 质量门禁

本文档记录 XUGOU 在提交、合并或部署前建议执行的检查命令。

## 推荐顺序

### 1. 安装依赖

```bash
pnpm install
```

### 2. 生成数据库迁移

当 `backend/src/db/schema.ts` 有变更时执行：

```bash
pnpm --dir backend run db:generate
pnpm --dir backend run generate:migrations
```

如果生成了新的 SQL 迁移，需要确认：

- SQL 文件已提交到 `backend/drizzle/`。
- `backend/drizzle/meta/_journal.json` 和对应 snapshot 已提交。
- `backend/src/db/generated-migrations.ts` 已更新。

### 3. 前端静态检查

```bash
pnpm --dir frontend run lint
```

目标结果：

- `0` error
- `0` warning

### 4. Agent 测试

```bash
cd agent && go test ./...
```

### 5. API 契约测试

```bash
pnpm run test:contracts
```

该命令会校验主要写接口的 zod schema，覆盖注册/登录、监控创建、Agent 注册与上报、状态页配置、通知设置和用户创建等请求结构。

### 6. 完整构建

```bash
pnpm run build
```

该命令会依次执行：

- 后端 Drizzle schema 迁移生成检查
- 前端 TypeScript build 和 Vite build
- 后端 TypeScript build
- 汇总迁移文件生成

## 部署前检查

部署前至少确认：

- `pnpm run build` 已通过。
- `pnpm --dir frontend run lint` 已通过。
- `pnpm run test:contracts` 已通过。
- `cd agent && go test ./...` 已通过。
- 如果新增迁移，已在本地 SQLite 或测试 D1 环境验证关键 SQL 可执行。
- 生产环境如需限制来源，已配置 `ALLOWED_ORIGINS`。
- 如需请求日志，显式设置 `LOG_REQUESTS=true`；默认生产环境不打印普通请求日志。
